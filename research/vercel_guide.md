# Vercel 入門ガイド

> 非エンジニアのための、はじめてのWebサイト公開！個人開発で使えるデプロイ入門

---

## はじめに：Vercel って何？

「Vercel（ヴァーセル）」は、**作ったウェブサイトを世界に公開するためのサービス**です。

### 🎯 イメージで理解しよう

Vercelは「**無料で使えるレストランの営業担当**」のようなものです。

```
あなたが作った料理（ウェブサイト）
         ↓
    Vercelが紹介
         ↓
    世界中のお客さんに届く
```

従来は、サーバーを用意して設定して…と大変でしたが、Vercelを使うと**ボタン1つで公開**できます。

### 💡 用語解説

| 用語 | 意味 |
|------|------|
| **デプロイ** | ウェブサイトを公開すること |
| **ホスティング** | ウェブサイトを置いておく場所 |
| **リポジトリ** | ファイルを保管する場所（GitHub） |
| **ビルド** | プログラムを実行可能な形に変換すること |

### 🌟 Vercelの主な特徴

| 特徴 | メリット |
|------|----------|
| **無料から使える** | 個人開発ならコストゼロ |
| **自動デプロイ** | コードを更新すると自動で公開 |
| **高速表示** | 世界中で速く表示される |
| **HTTPS対応** | 安全な通信が自動で有効 |

## 1. アカウント作成

### 1-1. Vercel とは？

**一言でいうと：クリックするだけでウェブサイトを公開してくれるサービス**

- 登録は無料
- GitHubと連携して超簡単
- コードを更新すると自動で公開される

### 1-2. アカウント作成手順

#### ステップ1：Vercelにアクセス

ブラウザで以下のURLを開きます：

```
https://vercel.com
```

#### ステップ2：サインアップ

1. 画面右上の「Sign Up」をクリック
2. GitHubアカウントでログイン（推奨）

> **💡 なぜGitHubが推奨なの？**
> - 1つのアカウントで管理できる
> - コードの管理と公開が一体化
> - 自動デプロイが簡単に設定できる

#### ステップ3：基本情報の入力

| 項目 | 入力内容 |
|------|----------|
| 名前 | 好きな名前（ニックネームでOK） |
| メールアドレス | 連絡が取れるメールアドレス |

> **📝 用語解説**
> - **GitHub（ギットハブ）**：プログラムのコードを保管・管理するサービス
> - **サインアップ**：新規登録のこと

#### ステップ4：ワークスペースの作成

- 「Individual」を選択（個人開発の場合）
- チーム開発でなければ「Hobby」プランでOK

```
┌─────────────────────────────────────┐
│  Choose your plan                   │
├─────────────────────────────────────┤
│  ○ Hobby    （無料、推奨）          │
│  ○ Pro      （有料）                │
│  ○ Enterprise（企業向け）           │
└─────────────────────────────────────┘
```

## 2. プロジェクトの連携

### 2-1. リポジトリの準備（GitHub）

Vercelで公開するには、**コードをGitHubに置く必要があります**。

#### GitHubにリポジトリを作成する手順

1. GitHubにログイン
2. 画面右上の「+」をクリック
3. 「New repository」を選択
4. 以下を入力：
   - **Repository name**：プロジェクト名（例：`my-first-site`）
   - **Description**：説明（任意）
   - **Public**：誰でも見れる
   - **Private**：自分だけが見れる

5. 「Create repository」をクリック

> **📝 用語解説**
> - **リポジトリ（レポジトリ）**：ファイルを保管する場所のこと
> - **パブリック**：誰でも閲覧可能
> - **プライベート**：自分だけが閲覧可能

#### ローカルのファイルをGitHubにアップする方法

```bash
# 1. Gitの初期化
git init

# 2. 全てのファイルを追加
git add .

# 3. コミット（変更を保存）
git commit -m "最初のコミット"

# 4. GitHubにアップ
git push -u origin main
```

> **💡 用語解説**
> - **コミット**：変更点を記録すること
> - **プッシュ**：GitHubにアップロードすること

### 2-2. Vercelへのインポート手順

#### ステップ1：Vercelダッシュボードを開く

```
https://vercel.com/dashboard
```

#### ステップ2：新しいプロジェクトを作成

1. 「Add New...」をクリック
2. 「Project」を選択

```
┌─────────────────────────────────────┐
│  Vercel ダッシュボード               │
├─────────────────────────────────────┤
│  [+ Add New...]  ↓                  │
│  ┌────────────────┐                 │
│  │ Project        │                 │
│  │ Database       │                 │
│  │ Team           │                 │
│  └────────────────┘                 │
└─────────────────────────────────────┘
```

#### ステップ3：GitHubと連携

1. 「Import Git Repository」をクリック
2. GitHubのアカウントを認証
3. リポジトリ一覧から公開したいものを選択
4. 「Import」をクリック

```
┌─────────────────────────────────────┐
│  Import Git Repository              │
├─────────────────────────────────────┤
│  検索: [my-first-site]              │
│                                     │
│  ✓ my-first-site        [Import]   │
│  ✓ another-project                   │
└─────────────────────────────────────┘
```

### 2-3. ビルド設定の基本

#### プロジェクトの設定画面

インポートすると、設定画面が表示されます。

| 項目 | 説明 | 通常の設定 |
|------|------|-----------|
| **Project Name** | プロジェクト名 | 自動で入力 |
| **Framework Preset** | フレームワークの種類 | 自動検出 |
| **Root Directory** | ルートディレクトリ | 通常は空欄 |
| **Build Command** | ビルドコマンド | 自動検出 |
| **Output Directory** | 出力先ディレクトリ | 自動検出 |

> **📝 用語解説**
> - **フレームワーク**：ウェブサイトを作るための道具セット（Next.js等）
> - **ルートディレクトリ**：プロジェクトの一番上の階層
> - **ビルドコマンド**：プログラムを実行可能な形にする命令

#### 自動検出されるフレームワーク

Vercelは以下のフレームワークを自動検出します：

- Next.js
- React
- Vue.js
- Astro
- Svelte
- その他多数

設定が完了したら「Deploy」ボタンをクリック！

## 3. デプロイ手順

### 3-1. デプロイとは何か

**デプロイ（Deploy）**：作ったウェブサイトを**世界中の人に見える状態にすること**。

イメージで理解：

```
【開発中】自分のパソコンの中だけ
         ↓ デプロイ
【公開中】世界中の誰でもアクセス可能
```

### 3-2. 自動デプロイの仕組み

Vercelの最大の特徴は、**コードを更新すると自動で公開される**ことです。

```
┌────────────┐
│ あなたが    │
│ コードを更新│
└─────┬──────┘
      │ Git Push
      ▼
┌────────────┐
│  GitHub    │ 変更を検知
└─────┬──────┘
      │ 通知
      ▼
┌────────────┐
│  Vercel    │ 自動ビルド＆デプロイ
└─────┬──────┘
      │
      ▼
┌────────────┐
│ ウェブサイト │ 自動更新！
│  が更新される │
└────────────┘
```

> **💡 すごいところ**
> - 寝ている間に自動で公開
> - 手動でアップロードする必要なし
> - 過去のバージョンに即座に戻せる

## 4. 環境変数の設定

### 4-1. 環境変数とは何か

**環境変数（かんきょうへんすう）**：**APIキーなどの秘密情報を安全に管理する仕組み**です。

イメージで理解：

```
【環境変数を使わない場合】
コードの中にAPIキーを書く
→ 誰でも見れてしまう！危険！

【環境変数を使う場合】
別の場所に秘密情報を保管
→ コードから参照だけする
→ 安全！
```

### 4-2. 環境変数の設定方法

#### ステップ1：プロジェクト設定を開く

1. Vercelダッシュボードを開く
2. プロジェクトを選択
3. 「Settings」タブをクリック
4. 「Environment Variables」を選択

```
┌─────────────────────────────────────┐
│  Environment Variables              │
├─────────────────────────────────────┤
│  [+ Add New]                        │
│                                     │
│  Key                Value           │
│  ───────────────────────────────   │
│  API_KEY            ***********    │
│  DATABASE_URL       ***********    │
│  SECRET_KEY         ***********    │
└─────────────────────────────────────┘
```

#### ステップ2：環境変数を追加

1. 「Add New」をクリック
2. 以下を入力：
   - **Key**：変数名（例：`API_KEY`）
   - **Value**：値（例：`sk-proj-xxxxx`）
3. 「Save」をクリック

> **📝 入力例**
> ```
> Key:   OPENAI_API_KEY
> Value: sk-proj-abcdefghijklmnopqrstuvwxyz123456
> ```

#### ステップ3：環境変数を使う（コード側）

```javascript
// Next.jsの場合
const apiKey = process.env.OPENAI_API_KEY;

// データを取得
const response = await fetch('https://api.example.com', {
  headers: {
    'Authorization': `Bearer ${apiKey}`
  }
});
```

> **💡 用語解説**
> - **APIキー**：サービスを利用するための秘密の鍵
> - **プロセス（process）**：実行中のプログラム
> - **プロセス環境変数（process.env）**：プログラムから読める環境変数

### 4-3. セキュリティの重要性

#### 絶対にやってはいけないこと

```javascript
// ❌ 危険！絶対にやめて！
const apiKey = "sk-proj-xxxxxxxxxxxx";

// GitHubにアップすると世界中に公開されます！
```

```javascript
// ✅ 正しい方法
const apiKey = process.env.API_KEY;
// 環境変数から読み込む
```

#### 環境変数のベストプラクティス

| ルール | 説明 |
|--------|------|
| **コードに書かない** | 環境変数を使う |
| **.gitignoreに追加** | .envファイルはアップしない |
| **必要なだけ設定** | 使わない変数は削除 |
| **定期的に更新** | APIキーは定期的に変更 |

> **📝 用語解説**
> - **.gitignore（ギットイグノア）**：Git管理から除外するファイルのリスト
> - **.envファイル**：環境変数を記述するファイル

## 5. カスタムドメイン

### 5-1. ドメインとは何か

**ドメイン**：ウェブサイトの**住所**のことです。

イメージで理解：

```
【家の住所】
東京都渋谷区○○1-2-3

【ウェブサイトの住所】
https://my-site.com
```

```
┌─────────────────────────────────────┐
│  URLの構成                          │
├─────────────────────────────────────┤
│  https://   my-site.com   /page     │
│     └─┬──┘      └───┬───┘    └┬┘   │
│   プロコル      ドメイン    パス    │
│   (https)     (住所)      (階層)    │
└─────────────────────────────────────┘
```

> **💡 用語解説**
> - **URL**：ウェブサイトの住所全体
> - **ドメイン**：サイトの名前（例：google.com）
> - **プロトコル**：通信のルール（https）

### 5-2. カスタムドメインの設定方法

#### ドメインを取得する

まず、ドメインを購入する必要があります。

| サービス | 特徴 |
|----------|------|
| **お名前.com** | 日本で人気 |
| **ムームードメイン** | 安価 |
| **Google Domains** | 簡単（※サービス終了予定） |
| **Cloudflare Registrar** | 原価販売 |

#### ステップ1：Vercelでドメインを追加

1. Vercelダッシュボードを開く
2. プロジェクトを選択
3. 「Settings」タブをクリック
4. 「Domains」を選択
5. ドメインを入力して「Add」をクリック

```
┌─────────────────────────────────────┐
│  Domains                            │
├─────────────────────────────────────┤
│  Enter a domain:                    │
│  [my-site.com              ] [Add]  │
│                                     │
│  ✓ my-site.com                     │
│    A Records                        │
│    76.76.21.21                      │
└─────────────────────────────────────┘
```

#### ステップ2：DNS設定

ドメイン業者のサイトでDNS設定を変更します。

```
【DNS設定例】
タイプ    名前        値
A        @           76.76.21.21
A        www         76.76.21.21
CNAME    www         my-site.vercel.app
```

> **📝 用語解説**
> - **DNS（ディーエヌエス）**：ドメインとIPアドレスを結びつける電話帳
> - **Aレコード**：ドメインをIPアドレスに結びつける設定
> - **CNAME（シーネーム）**：ドメインを別のドメインに結びつける設定

#### ステップ3：設定を確認

Vercelの「Domains」画面で「Valid Configuration」になれば完了です！

```
┌─────────────────────────────────────┐
│  ✓ Valid Configuration              │
├─────────────────────────────────────┤
│  my-site.com                        │
│  A Records                          │
│  76.76.21.21 ← 正しく設定されてます   │
└─────────────────────────────────────┘
```

### 5-3. SSL証明書の自動設定

Vercelを使うと、**SSL証明書が自動で設定されます**。

#### SSLとは？

**SSL（エスエスエル）**：通信を暗号化する仕組み。

```
【SSLがない場合】
通信内容が丸見え（危険！）

【SSLがある場合】
通信内容が暗号化（安全！）
```

#### 見分け方

| 項目 | SSLなし | SSLあり |
|------|---------|---------|
| URL | `http://` | `https://` |
| ブラウザ | 警告表示 | 鍵マーク 🔒 |

> **💡 すごいところ**
> - VercelはSSL証明書を無料で自動発行
> - 更新も自動
> - 設定は不要

## 6. よくあるミスと対処法

| ミス | 原因 | 対処法 |
|------|------|--------|
| デプロイが失敗する | ビルドエラー | ログを確認してエラーを修正 |
| 環境変数が効かない | 本番環境に設定していない | 「Environment Variables」で設定 |
| ドメインが反映されない | DNS設定が間違っている | DNS設定を再確認 |
| 更新が反映されない | キャッシュの残り | `Cmd + Shift + R` でリロード |

### 💡 トラブルシューティング

#### デプロイが失敗した場合

1. **デプロイログを確認**
   ```
   Vercelダッシュボード
   → 該当のデプロイを選択
   → 「Build Logs」を確認
   ```

2. **エラーメッセージを読む**
   ```
   例：Error: Cannot find module 'xxx'
   → 必要なパッケージがインストールされていない
   → npm install でインストール
   ```

3. **ローカルでテスト**
   ```bash
   # ローカルでビルドして確認
   npm run build
   ```

#### 環境変数が動かない場合

1. **変数名の確認**
   ```javascript
   // 大文字・小文字を区別
   API_KEY ≠ api_key
   ```

2. **再デプロイ**
   ```
   環境変数を変更したら
   必ず再デプロイが必要
   ```

3. **各環境の設定を確認**
   ```
   Production（本番）   ← ここに設定している？
   Preview（プレビュー） ← ここに設定している？
   Development（開発）   ← ここに設定している？
   ```
