個人開発を始めたいけど、デザインがわからない...そんな悩みはありませんか？

## 第1章 UIデザインの基本

### 1-1 Figmaとは何か？

Figmaは、ブラウザ上で動作するデザインツールです。

**ポイント**
- インストール不要（ブラウザだけで動く）
- 無料で使える（個人利用の場合）
- スマホやタブレットからもアクセス可能

**身近な例え**
- PowerPointでプレゼン資料を作るのと似ています
- ただしFigmaは「Webサイトやアプリの画面」を作るためのツールです

### 1-2 アカウント作成

1. [figma.com](https://www.figma.com) にアクセス
2. 「Sign up」をクリック
3. Googleアカウントやメールアドレスで登録
4. プラン選択画面では「Starter（無料）」を選択

### 1-3 基本操作

| 操作 | 方法 |
|------|------|
| 新規作成 | 左上の「+」ボタン → 「Design file」 |
| 図形の追加 | 右上の「Shape」から選択 |
| テキスト追加 | 「T」アイコンをクリック |
| 色変更 | 右側のパネル「Fill」をクリック |
| コピー | `Ctrl+C`（Mac: `Cmd+C`） |
| 貼り付け | `Ctrl+V`（Mac: `Cmd+V`） |

### 1-4 なぜデザインツールが必要なのか？

**イメージだけで作る問題点**
- コーディング中に「見た目」を気にする必要がある
- レイアウトの調整に時間がかかる
- 完成イメージがわかりにくい

**デザインを先に作るメリット**
- コードに集中できる（デザインは決まっているから）
- 完成イメージが明確
- 変更容易（コードを書き直すよりはるかに簡単）

## 第2章 コンポーネントのデザイン

### 2-1 コンポーネントとは？

**コンポーネント = 使い回せる部品**

ボタン、カード、入力欄など、アプリの中で何度も使うパーツを「コンポーネント」として登録しておくと、一箇所を修正するだけで全てのボタンが変更されます。

**日常的な例え**
- Wordの「テンプレート」や「スタイル」のようなもの
- 一度作ったボタンをコピーして、色やサイズだけ変えて使い回せる

### 2-2 コンポーネントの作成方法

**ステップ**

1. 図形やテキストでボタンを作る
2. 作ったパーツを選択
3. 右上の「Create component」アイコン（⚡）をクリック
4. コンポーネント名を付ける（例: 「Primary Button」）

**実際の画面構成**
```
┌─────────────────────────────────┐
│     [クリックして送信]           │ ← 通常のボタン
└─────────────────────────────────┘
           ↓ コンポーネント化
┌─────────────────────────────────┐
│   ⚡ Primary Button              │ ← コンポーネント（部品）
└─────────────────────────────────┘
```

### 2-3 バリアントの作成

バリアント = 同じコンポーネントの「バリエーション」

**例：ボタンのサイズ違い**
- Small（小）
- Medium（中）
- Large（大）

**作成手順**

1. コンポーネントを選択
2. 右側パネルの「Variants」をクリック
3. 「+ Add variant」でバリエーション追加
4. 各バリアントのサイズや色を調整

**バリアントの活用例**

| 種類 | バリアント |
|------|-----------|
| サイズ | Small, Medium, Large |
| 状態 | Default, Hover, Pressed, Disabled |
| 色 | Primary, Secondary, Danger |

## 第3章 デザインシステムの構築

### 3-1 デザイントークンとは？

**デザインントークン = デザインの共通設定**

色、フォント、余白などを「トークン」として管理することで、アプリ全体で統一感のあるデザインを作れます。

**分かりやすい例え**
- 家具のセット購入で「同じ色・同じシリーズ」の家具を揃えるようなもの
- 色がバラバラだと違和感があるのと同じで、デザインも統一感が大切

### 3-2 トークンで管理するもの

**カラートークン**
```
Primary   → #3B82F6（青） メインボタン、リンク
Secondary → #6B7280（灰） 補足情報
Success   → #10B981（緑） 成功メッセージ
Error     → #EF4444（赤） エラーメッセージ
```

**フォントトークン**
```
Heading   → 24px、太字
Body      → 16px、標準
Caption   → 12px、細字
```

**スペース（余白）トークン**
```
XS  → 4px
S   → 8px
M   → 16px
L   → 24px
XL  → 32px
```

### 3-3 スタイルライブラリの活用

Figmaには「Local Styles」という機能があり、色やテキストスタイルを保存しておけます。

**登録手順**

1. オブジェクトを選択
2. 右パネルで「Fill」や「Text」の4点アイコンをクリック
3. 「+」で新規スタイル作成
4. 名前を付ける（例: `Color/Primary`）

**スタイルの命名規則（推奨）**

```
Color/Primary
Color/Secondary
Text/Heading
Text/Body
```

### 3-4 一貫性のあるデザインを作るコツ

**ルールを決める**
- 使う色は最大5色まで
- フォントは2種類まで
- 余白は8px単位で揃える

**チェックリスト**
- [ ] 同じ用途のボタンは同じ見た目か
- [] 見出しのサイズが統一されているか
- [] 余白の間隔が揃っているか
- [] 色の意味が統一されているか

## 第4章 コードへの実装方法

### 4-1 FigmaからTailwind CSSへの変換

Tailwind CSSは、HTMLのクラス名でデザインを指定するCSSフレームワークです。

**FigmaとTailwindの対応**

| Figma | Tailwind CSS |
|-------|-------------|
| 色: #3B82F6 | `bg-blue-500` |
| 余白: 16px | `p-4` |
| フォント: 24px | `text-2xl` |
| 角丸: 8px | `rounded-lg` |

**変換例**

Figmaのデザイン：
```
■ 青色のボタン
■ 余白 16px
■ 文字 16px、太字
```

↓ Tailwind CSSコード

```html
<button class="bg-blue-500 px-4 py-2 text-base font-bold rounded-lg">
  ボタン
</button>
```

### 4-2 プラグインの活用

Figmaには、デザインからコードを自動生成するプラグインがあります。

**おすすめプラグイン**

| プラグイン | 用途 |
|-----------|------|
| Tailwind CSS | FigmaのデザインからTailwindクラスを取得 |
| Anima | HTML/CSS/Reactのコードを生成 |
| Builder.io | Visualからコードへ変換 |

**Tailwind CSSプラグインの使い方**

1. Figmaで「Resources」→「Plugins」→「Browse plugins」
2. 「Tailwind CSS」を検索してインストール
3. デザインを選択してプラグイン実行
4. Tailwindクラスがコピーされる

### 4-3 デザインとコードのギャップを埋める方法

**よくある問題**

| 問題 | 原因 | 対策 |
|------|------|------|
| 色が違う | カラープロファイルの違い | Hexコードを直接指定 |
| レイアウトが崩れる | 固定幅 vs レスポンシブ | Auto Layoutを活用 |
| フォントが違う | Webフォント未導入 | Google Fontsなどを導入 |

**Auto Layoutの活用**

FigmaのAuto Layoutは、CSSのFlexboxに似ています。

- 要素の間隔を自動調整
- レスポンシブなレイアウトに対応
- コードへの変換がスムーズ

## 第5章 よくあるミスと対処法

### ミス1: コンポーネント化し忘れ

**問題**: 同じボタンを何度もコピーして修正が大変

**対処**: 使い回すパーツはすべてコンポーネント化

### ミス2: 色の使いすぎ

**問題**: 色が多すぎて統一感がない

**対処**: メイン色1色、サブ色2色、グレー2色に限定

### ミス3: 余白がバラバラ

**問題**: 間隔が不揃いで見にくい

**対処**: 8px単位で統一（4px, 8px, 16px, 24px...）

### ミス4: 画像サイズが合わない

**問題**: 画像が伸び縮みして見にくい

**対処**: 「Fill」ではなく「Fit」や「Crop」を使用

## 第6章 サンプルコード

### サンプル1: シンプルなボタン

**Figmaデザイン**
- 幅: 120px、高さ: 40px
- 背景: 青 (#3B82F6)
- 文字: 白、中央揃え

**Tailwind CSS**
```html
<button class="w-30 h-10 bg-blue-500 text-white rounded-lg">
  送信
</button>
```

### サンプル2: カードコンポーネント

**Figmaデザイン**
- 背景: 白
- 角丸: 12px
- 影: 薄いシャドウ
- 余白: 24px

**Tailwind CSS**
```html
<div class="bg-white rounded-xl shadow-md p-6">
  <h2 class="text-xl font-bold mb-2">カードタイトル</h2>
  <p class="text-gray-600">カードの説明文</p>
</div>
```

### サンプル3: 入力フォーム

**Figmaデザイン**
- 入力欄: 枠線、角丸8px
- ラベル: 上に配置
- エラー時: 赤い枠線

**Tailwind CSS**
```html
<div>
  <label class="block text-sm font-medium mb-1">メールアドレス</label>
  <input
    type="email"
    class="w-full border border-gray-300 rounded-lg px-4 py-2 focus:ring-2 focus:ring-blue-500"
    placeholder="example@email.com"
  />
</div>
```

## 用語集

| 用語 | 読み方 | 意味 |
|------|--------|------|
| コンポーネント | Component | 使い回せる部品のこと |
| バリアント | Variant | 同じ部品のバリエーション |
| デザイントークン | Design Token | デザインの共通設定 |
| Auto Layout | オートレイアウト | 要素の配置を自動化する機能 |
| プラグイン | Plugin | Figmaに機能を追加するツール |
| Tailwind CSS | テイルウィンド | クラス名でデザインするCSSフレームワーク |
| Flexbox | フレックスボックス | CSSのレイアウト方法の一つ |
| レスポンシブ | Responsive | 画面サイズに合わせてレイアウトを変えること |
| ヘックスカラー | Hex Color | 色を表すコード（例: #3B82F6） |
